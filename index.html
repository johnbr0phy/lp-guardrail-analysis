Competitor Analysis

Landing Page Module Creation, Saving & Guardrails

HubSpot CMS Hub

Module Creation & Saving

  • Custom Modules: Developers create reusable modules (HTML, HubL, CSS, JS) in the Design Manager. Offers high flexibility.
  • Module Library: Saved modules are organized and accessible via folders for reuse.
  • Saved Sections: Simpler, user-facing feature to save pre-built sections directly from the page editor (less complex than full modules).
  • Themes: Control overall page structure and styling templates.

Guardrails & Restrictions

  • Field Locking: Developers can lock specific content fields within a module, preventing edits by content creators (e.g., lock button style but allow text change).
  • Style Fields: Limit style choices (colors, fonts, spacing) to pre-defined options set in the theme or module settings, enforcing brand consistency.
  • Content Input Rules: Define field types (text, image, boolean), set character limits, and provide default values.
  • Global Modules/Sections: Edits to a global element update everywhere it's used (ideal for headers/footers).
  • Theme Settings: Centralized control over global styles applied across pages.
  • Permissions: Standard HubSpot user roles can restrict access to design tools or specific content areas.

Note: Guardrail effectiveness often depends on how meticulously the developer builds the module and configures its options.

Knak

Module Creation & Saving

  • Centralized Module Library: Core feature. Modules are typically built/approved by designated admins or Knak services to ensure strict brand compliance.
  • Drag-and-Drop Assembly: Standard users build pages primarily by selecting and arranging pre-approved modules from the library.
  • Platform Focus: Built specifically for scalable, brand-controlled email and landing page creation in enterprise environments.

Guardrails & Restrictions

  • Highly Granular Locking: Knak excels here. Admins can lock almost any element within a module (text, images, links, styling, layout structure).
  • Defined Editable Zones: Only explicitly designated areas within a module can be modified by end-users.
  • Strict Brand Kit Enforcement: Centrally managed brand assets (colors, fonts, logos, spacing) are rigorously enforced. Users select from approved options only.
  • Role-Based Access Control: Define which user roles can access specific modules or have specific editing rights.
  • Integration Controls: Ensures correct setup and data flow with Marketing Automation Platforms (MAP) / CRMs.
  • Approval Workflows: Often includes workflows for reviewing and approving content before publishing.

Example: Knak's emphasis on a locked-down module library (conceptual).

Note: Knak's primary value proposition is its robust, enterprise-grade guardrails and brand control system.

Unbounce

Module Creation & Saving

  • Saved Page Sections: Users can save sections they build within the editor for reuse across different pages (similar to HubSpot's Saved Sections).
  • Global Blocks: Specific blocks (like headers/footers) can be designated as global for consistent updates.
  • Templates: Create and reuse full landing page templates to standardize layouts.
  • Copy/Paste Functionality: Allows copying elements or sections between pages.

Guardrails & Restrictions

  • Basic Style Guide: Define default colors and fonts for text elements within the builder, promoting consistency.
  • Templates as Foundation: Using templates provides an initial structure and style baseline.
  • User Roles (Enterprise): Offers roles (Admin, Author, Viewer) limiting actions like publishing, deleting, or accessing account settings.
  • Less Granular Content Locking: Generally provides more freedom to users within sections compared to Knak or well-built HubSpot modules. Less focus on locking individual content fields *within* a saved section.
  • Script Manager Control: Admins can manage global scripts, adding some technical control.

Example: Unbounce builder with saved sections concept (conceptual).

Note: Unbounce prioritizes speed and conversion flexibility for marketers. Guardrails exist, particularly on higher tiers, but might be less restrictive on content editing within modules/sections compared to Knak.
